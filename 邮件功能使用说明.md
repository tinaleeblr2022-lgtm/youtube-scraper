# 邮件发送功能使用说明

## 功能概述
在现有 YouTube 频道抓取工具基础上，新增了自动化邮件发送功能，支持批量发送个性化邮件。

## 启动步骤

### 1. 启动后端服务
```bash
cd /Users/liting/youtube-scraper/backend
npm start
```

### 2. 打开前端页面
在浏览器中打开：
```
file:///Users/liting/youtube-scraper/frontend/public/index.html
```

## 功能使用指南

### 切换功能标签
- 页面顶部有两个标签：「YouTube抓取」和「邮件发送」
- 点击对应标签即可切换功能

### 邮件发送流程

#### 第一步：配置 SMTP 信息
在「SMTP配置��区域填写：
- **发件邮箱**：你的邮箱地址（如：your@gmail.com）
- **SMTP服务器**：邮箱服务商的 SMTP 地址
- **端口**：通常为 587（TLS）或 465（SSL）
- **密码/授权码**：邮箱密码或应用专用密码

**常用邮箱 SMTP 配置**：
- **Gmail**：
  - 服务器：smtp.gmail.com
  - 端口：587
  - 需要开启「应用专用密码」
- **QQ邮箱**：
  - 服务器：smtp.qq.com
  - 端口：587
  - 需要开启「SMTP服务」并获取授权码
- **163邮箱**：
  - 服务器：smtp.163.com
  - 端口：465
  - 需要开启「SMTP服务」并获取授权码

点击「测试连接」验证配置是否正确。

#### 第二步：编辑邮件内容
在「邮件内容」区域：
- **邮件标题**：输入邮件主题，支持使用 `{客户名称}` 变量
- **邮件正文**：输入邮件内容，使用 `{客户名称}` 会自动替换为 Excel 中的客户名称
- **发送间隔**：设置每封邮件之间的发送时间（秒），建议设置为 2-5 秒避免被判定为垃圾邮件

**模板管理**：
- 点击「保存模板」可以将当前邮件内容保存为模板
- 从下拉菜单中选择已有模板快速加载
- 点击「删除模板」移除不需要的模板

#### 第三步：上传客户列表
点击上传区域或拖拽上传 Excel 文件（.xlsx格式）

**Excel 文件格式要求**：
- 文件必须是 .xlsx 格式
- 至少包含两列：客户名称、邮箱地址
- 工具会自动识别列名（支持多种命名方式）：
  - 客户名称列：客户、名称、姓名、name、customer
  - 邮箱列：邮箱、邮件、email、mail

**示例 Excel 结构**：
| 客户名称 | 邮箱地址 |
|---------|---------|
| 张三 | zhangsan@email.com |
| 李四 | lisi@email.com |

上传成功后会显示客户列表表格，可以勾选/取消勾选要发送的客户。

#### 第四步：发送邮件
1. 确认 SMTP 配置正确
2. 确认邮件内容完整
3. 选择要发送的客户（默认全选）
4. 点击「发送邮件」按钮

发送过程中会显示实时进度：
- 总数、已发送、成功、失败的统计数据
- 进度条显示发送百分比

#### 第五步：查看发送结果
发送完成后会显示详细结果列表：
- ✅ 成功：邮件已发送
- ❌ 失败：显示失败原因（如邮箱地址错误、连接失败等）

对于失败的邮件，可以点击「重试失败邮件」按钮重新发送。

## 注意事项

1. **SMTP 配置安全**：
   - 不要在公共电脑上保存邮箱密码
   - 建议使用邮箱服务商提供的「应用专用密码」而非邮箱密码

2. **发送频率控制**：
   - 建议设置 2-5 秒的发送间隔
   - 大量发送时避免一次性发送过多（建议每次不超过 100 封）

3. **垃圾邮件风险**：
   - 确保收件人同意接收邮件
   - 避免使用敏感词汇
   - 提供退订方式

4. **邮箱服务商限制**：
   - 不同邮箱服务商有每日发送量限制
   - 超过限制可能被临时封禁
   - Gmail：每天约 500 封
   - QQ邮箱：每天约 200 封
   - 163邮箱：每天约 200 封

## 常见问题

**Q: SMTP 连接测试失败怎么办？**
A: 检查以下项目：
- SMTP 服务器地址和端口是否正确
- 是否开启了 SMTP 服务（需要登录邮箱设置中开启）
- 是否使用了授权码而非登录密码（QQ/163邮箱需要）
- 防火墙是否阻止了连接

**Q: 发送失败显示 "authentication failed"？**
A: 认证失败，通常是密码错误，建议：
- 使用邮箱的应用专用密码
- 检查是否开启了「允许不够安全的应用」（Gmail）

**Q: 发送失败显示 "invalid email address"？**
A: 邮箱地址格式错误，检查 Excel 中的邮箱地址是否正确。

**Q: 邮件发送成功但收件人没收到？**
A: 可能的原因：
- 邮件被收件箱服务商拦截到垃圾邮件箱
- 收件人邮箱地址错误
- 发送频率过快被服务商限制

## 技术架构

- **后端**：Node.js + Express
- **邮件发送**：Nodemailer
- **Excel 处理**：XLSX.js
- **前端**：原生 JavaScript + HTML5

## 数据隐私

- 所有数据仅在本地处理
- SMTP 配置信息不会存储
- 邮件模板保存在浏览器本地存储中
- 不会收集或上传任何用户数据
