## 🎯 核心特性

### 智能列名识别
系统会自动识别 Excel 中的列名，无需严格固定列名：

**客户名称列支持**：
- 客户、名称、姓名
- name、customer

**邮箱列支持**：
- 邮箱、邮件
- email、mail

### 变量替换系统
邮件模板中支持 `{客户名称}` 变量，发送时自动替换为真实客户名称：
```
尊敬的 {客户名称}，您好！
↓
尊敬的 张三，您好！
```

### 模板管理系统
- 保存多个邮件模板
- 模板存储在浏览器本地存储中
- 可随时加载、编辑、删除模板
- 页面刷新后模板仍然保留

### 发送控制机制
- 可调节发送间隔（1-60秒）
- 实时显示发送进度
- 详细的成功/失败统计
- 失败邮件支持一键重试

## 🔧 技术架构

### 后端技术栈
- **Node.js + Express** - Web 服务框架
- **Nodemailer** - 邮件发送核心库
- **Multer** - 文件上传中间件
- **XLSX** - Excel 文件解析

### 前端技术栈
- **原生 JavaScript** - 无需前端框架
- **File API** - 文件拖拽上传
- **Local Storage** - 模板本地存储
- **Fetch API** - 与后端通信

### API 接口

**1. 测试SMTP连接**
```
POST /api/email/test-smtp
Body: { smtpConfig }
```

**2. 解析Excel文件**
```
POST /api/email/parse-excel
Body: multipart/form-data (file)
```

**3. 发送邮件**
```
POST /api/email/send
Body: { smtpConfig, recipients, subject, body, interval }
```

**4. 重试失败邮件**
```
POST /api/email/retry
Body: { smtpConfig, failedResults, subject, body, interval }
```

## ⚠️ 注意事项

1. **SMTP 配置**：
   - QQ/163邮箱需要使用授权码
   - Gmail 需要开启两步验证并生成应用专用密码
   - 企业邮箱请联系管理员获取 SMTP 配置

2. **发送限制**：
   - Gmail：每天约 500 封
   - QQ邮箱：每天约 200 封
   - 163邮箱：每天约 200 封
   - 超过限制可能被临时封禁

3. **发送建议**：
   - 建议设置 2-5 秒的发送间隔
   - 每次发送不超过 100 封
   - 确保收件人同意接收邮件
   - 避免使用敏感词汇

## 🐛 已知限制
